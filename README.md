# Analyzing-E-Learning-Platform-Purchases-using-MySQL
SQL-based analysis of e-learning platform data to understand learner behavior, course performance, and revenue trends using joins, aggregations, and advanced queries.

📌 Project Overview

This project analyzes purchase data from an e-learning platform to understand learner behavior, course performance, and sales trends. A relational database was designed using MySQL, and SQL queries were used to extract meaningful business insights.

🎯 Objective

Analyze learner purchase patterns

Identify top-performing courses

Evaluate revenue and sales performance

Detect underperforming courses

🛠️ Tools & Technologies

MySQL

SQL (Joins, Aggregations, Subqueries, CTEs)

🗂️ Database Structure

The database consists of three main tables:

Learners – stores learner details

Courses – contains course information

Purchases – stores transaction data

🔍 Key SQL Concepts Used

Joins (INNER, LEFT, RIGHT)

Aggregations (SUM, COUNT, GROUP BY)

Subqueries & CTEs

Data filtering and grouping

📈 Analysis & Insights

1. Descriptive Analysis (What happened?)

Analyzed overall platform activity including learner purchases and course details

Calculated total purchases, quantities, and revenue

Provided a summary of how the platform is performing

2. Sales Performance Analysis (How is the business performing?)

Identified top-selling courses and high-revenue categories

Calculated total revenue and quantity sold

Highlighted the most profitable courses

3. Learner Behavior Analysis (How users behave?)

Analyzed total spending per learner

Identified learners purchasing across multiple categories

Discovered patterns in customer engagement and buying behavior

4. Gap Analysis (What is not performing?)

Identified courses with zero or low purchases

Highlighted underperforming courses

Suggested areas for improvement in pricing or visibility

📊 Key Insights

Certain courses generate higher revenue and demand

Some learners show high engagement by purchasing multiple courses

A few courses have no sales, indicating performance gaps

SQL analysis helps convert raw data into actionable insights

✅ Conclusion

This project demonstrates how SQL can be used to analyze structured data and generate business insights. By applying joins, aggregations, and advanced queries, the project provides valuable insights into learner behavior, sales performance, and business improvement opportunities.
